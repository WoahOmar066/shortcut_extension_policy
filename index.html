Privacy Policy

Custom Paste Command Extension

Last Updated: April 8, 2025

This Privacy Policy outlines the data practices of the Custom Paste Command Extension (hereinafter referred to as "the Extension," "we," "our," or "us"). This document is intended to provide transparent information about how the Extension collects, uses, stores, and protects user data. By installing and using the Extension, you acknowledge that you have read and understood this Privacy Policy.

1. Data Collection and Processing

1.1 Types of Data Collected

The Extension collects and processes the following limited categories of data:

a) Keyboard Input Monitoring: The Extension monitors keyboard events solely for the purpose of detecting when a user-defined keyboard shortcut has been pressed. This monitoring is performed locally within the browser environment and is limited to detecting specific key combinations that users have explicitly configured as shortcuts.

b) User-Defined Shortcuts: The Extension stores keyboard shortcut combinations that users have explicitly created and configured within the Extension.

c) User-Defined Text Snippets: The Extension stores text snippets that users have explicitly associated with their keyboard shortcuts for pasting purposes.

1.2 Technical Implementation Details

The keyboard event monitoring operates through the following mechanism:

a) When a webpage loads, the Extension injects a key listener script that monitors keyboard input events.

b) When keys are pressed, the Extension checks if the combination matches a saved shortcut.

c) If a match is found, the Extension pastes the corresponding text into the active field.

d) No keystroke data is logged, recorded, or transmitted beyond the immediate check against saved shortcuts.

1.3 Purpose Limitation

All data collection and processing activities are strictly limited to enabling the core functionality of the Extension. Specifically, data is collected and processed solely to:

a) Facilitate the detection of user-defined keyboard shortcuts when they are pressed.

b) Enable the insertion of corresponding text snippets into active text fields when shortcuts are detected.

c) Allow users to manage (create, view, edit, or delete) their custom shortcuts and text snippets.

2. Data Storage

2.1 Local Storage

All data collected by the Extension is stored exclusively in the user's browser using Chrome's local storage API. Key aspects of this storage approach include:

a) Device Limitation: Data is stored only on the user's device and is not synchronized across devices or browsers.

b) No Remote Storage: No user data is transmitted to or stored on any remote servers, including servers operated by us or by third parties.

c) Browser Control: The data stored by the Extension is subject to the security controls and limitations imposed by the Chrome browser and its local storage implementation.

2.2 Data Retention and Deletion

a) Duration of Storage: Data is retained in local storage for as long as the Extension remains installed or until the user manually deletes specific shortcuts.

b) User Control: Users have complete control over their data and can delete individual shortcuts or all data at any time through the Extension's interface.

c) Uninstallation: When the Extension is uninstalled, all associated data in the browser's local storage is permanently removed.

3. Data Sharing and Transmission

3.1 No Data Transmission

The Extension does not transmit any user data outside of the user's browser. Specifically:

a) No Internet Communication: The Extension does not send any collected data over the internet to any servers or third parties.

b) No Analytics: The Extension does not implement any form of analytics or telemetry that would transmit usage data.

c) No API Calls: The Extension does not make API calls to external services that would involve sharing user data.

3.2 Third-Party Access

a) No Third-Party Sharing: We do not share, sell, rent, trade, or otherwise disclose any user data to third parties, as we do not have access to or possession of such data in the first place.

b) No Third-Party Services: The Extension does not integrate with or utilize any third-party services that would have access to user data.

c) No Advertising: The Extension does not serve advertisements and does not share any information with advertising networks.

3.3 Legal Disclosure

As we do not collect or store any user data on our servers, we do not possess any user data that could be disclosed in response to legal requests such as subpoenas, court orders, or other legal processes.

4. Data Security

4.1 Technical Security Measures

The Extension implements appropriate technical measures to ensure the security of user data:

a) Local Processing: All data processing occurs locally within the user's browser, minimizing exposure to external threats.

b) Chrome API Security: The Extension relies on Chrome's built-in security mechanisms for local storage, which include isolation between different extensions and websites.

c) Minimal Permissions: The Extension requests only those permissions that are strictly necessary for its functionality, as detailed in the permission justifications section.

4.2 Data Access Controls

a) User Control: Only the user has access to their saved shortcuts and text snippets through the Extension's interface.

b) Developer Non-Access: As developers, we have no means to access individual users' data since it is stored exclusively on users' devices.

5. User Rights and Controls

5.1 Access and Control

Users have comprehensive rights regarding their data, including:

a) Transparency: Users can view all shortcuts and text snippets they have created through the Extension's popup interface at any time.

b) Modification: Users can edit or update their shortcuts and text snippets through the Extension's interface.

c) Deletion: Users can delete individual shortcuts or all data by using the Extension's interface or by uninstalling the Extension.

d) Installation Control: Users can choose to uninstall the Extension at any time, which will remove all associated data.

5.2 User Consent

The Extension operates on the basis of explicit user consent, as evidenced by:

a) Voluntary Installation: Users must actively choose to install the Extension.

b) Permission Approval: Users must approve the requested permissions during installation.

c) Explicit Configuration: Users must manually create and configure any shortcuts and text snippets.

6. Children's Privacy

The Extension is not directed at children under the age of 13 and does not knowingly collect personal information from children. The functionality of the Extension is general-purpose and not specifically designed for or targeted towards children.

a) Update the policy text in the Chrome Web Store listing

b) Update the version information for the Extension

c) Include a note about the change in the Extension's update notes

Users are encouraged to review this Privacy Policy periodically to stay informed about our data practices. Continued use of the Extension after any modifications to the Privacy Policy constitutes acceptance of those changes.

8. Permissions Justification

The Extension requires several permissions to function properly. Here is a detailed explanation of each permission and its necessity:

a) clipboardWrite: Required to insert custom text into text fields when shortcuts are activated. Without this permission, the Extension would be unable to paste the user's custom text snippets.

b) activeTab: Necessary to access the current tab when a shortcut is pressed, allowing the Extension to identify where to insert text and ensuring text is only pasted into the active text field or editor.

c) storage: Essential for saving custom keyboard shortcuts and their corresponding text snippets to Chrome's local storage, ensuring shortcuts persist between browser sessions.

d) scripting: Required to inject the key listener script into web pages, enabling the Extension to detect when custom shortcuts are pressed while browsing.

e) Host Permission (""): Necessary for the Extension to detect and respond to keyboard shortcuts on any website. This broad access is required for the Extension to function across the entire web rather than on a limited set of sites.
